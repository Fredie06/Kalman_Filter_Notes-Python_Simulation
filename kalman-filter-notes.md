## 1. 基本设定

### 1.1 一条核心铁律

卡尔曼滤波适用于**线性 + 高斯**系统：

$$
\text{高斯} \times \text{线性变换} = \text{仍然是高斯}
$$

对于高斯的简单理解就是高斯噪声，即噪声服从 **正态分布**。

初始高斯 → 经线性状态方程 → 先验仍是高斯 → 经线性观测方程 → 观测值仍是高斯 → 高斯先验 × 高斯观测值 = 后验也是高斯。链条一断（非线性 or 非高斯），经典 KF 不适用。

### 1.2 KF 家族速览

| | 高斯噪声 | 非高斯噪声 |
|---|---|---|
| **线性** | **KF**（最优） | KF 仍是 BLUE |
| **非线性** | EKF / UKF | 粒子滤波 |

- **EKF**：一阶泰勒展开，局部线性化
- **UKF**：不线性化，sigma points 采样穿非线性函数
- **粒子滤波**：放弃高斯假设，用带权粒子近似任意分布

## 2. 方程、符号与核心公式

### 2.1 两个基础方程

**状态方程**（系统如何随时间演化）：

$$
x_k = F x_{k-1} + B u_k + w_k, \quad w_k \sim \mathcal{N}(0, Q)
$$

**观测方程**（传感器如何看到状态）：

$$
z_k = H x_k + v_k, \quad v_k \sim \mathcal{N}(0, R)
$$

### 2.2 五个核心公式与数据流

卡尔曼滤波是一个**预测 → 更新**的循环。五个公式分两组：

| | 公式（多维） | 一维特例 | 做什么 |
|---|---|---|---|
| **① 预测** | $\hat{x}_k^- = F \hat{x}_{k-1} + B u_k$ | $\hat{x}^- = f \cdot \hat{x}_{k-1} + b \cdot u_k$ | 闭眼，按模型猜状态 |
| **② 预测** | $P_k^- = F P_{k-1} F^\top + Q$ | $P^- = f^2 \cdot P_{k-1} + Q$ | 闭眼后，更不确定了 |
| **③ 增益** | $K_k =\frac{ P_k^- H^\top}{H P_k^- H^\top + R} $ | $K = \frac{P^-}{P^- + R}$ | 算权重：谁方差小就多信谁 |
| **④ 更新** | $\hat{x}_k = \hat{x}_k^- + K_k (z_k - H \hat{x}_k^-)$ | $\hat{x} = \hat{x}^- + K(z - \hat{x}^-)$ | 睁眼，加权融合 |
| **⑤ 更新** | $P_k = (I - K_k H) P_k^-$ | $P = (1 - K) P^-$ | 看了传感器，信心恢复 |

**数据在五个公式间的流动：**

```
上一时刻后验 x̂ₖ₋₁, Pₖ₋₁
       │
       ▼
  ┌──────────────────┐
  │  预测步（①②）     │
  │  x̂ₖ⁻ = F x̂ₖ₋₁    │  ← 只用模型，不看传感器
  │  Pₖ⁻  = F Pₖ₋₁ Fᵀ + Q │
  └──────────────────┘
       │
       ▼  先验 x̂ₖ⁻, Pₖ⁻
       │
  ┌──────────────────┐
  │  更新步（③④⑤）   │
  │  Kₖ = ...        │  ← 拿传感器数据融合
  │  x̂ₖ = x̂ₖ⁻ + K(z-Hx̂⁻) │
  │  Pₖ = (I-KH)Pₖ⁻    │
  └──────────────────┘
       │
       ▼  后验 x̂ₖ, Pₖ
       │
       └──→ 成为下一时刻的输入
```

### 2.3 符号约定与全量表

| 符号 | 含义 |
|------|------|
| $x_k$ | **真实**状态向量（n×1），永远未知 |
| $\hat{x}_k$ | $x_k$ 的**后验估计**（最优估计），算法输出 |
| $\hat{x}_k^-$ | $x_k$ 的**先验估计**（未用观测修正，纯靠模型猜） |
| $F$ | 状态转移矩阵（n×n） |
| $B$ | 控制矩阵（n×p） |
| $u_k$ | 控制输入（p×1） |
| $w_k$ | 过程噪声，$w_k \sim \mathcal{N}(0, Q)$ |
| $Q$ | 过程噪声协方差矩阵（n×n） |
| $z_k$ | 传感器观测值（m×1） |
| $H$ | 观测矩阵（m×n），把状态映射到观测空间 |
| $v_k$ | 观测噪声，$v_k \sim \mathcal{N}(0, R)$ |
| $R$ | 观测噪声协方差矩阵（m×m） |
| $P_k$ | **后验**估计误差协方差，$P_k = \mathrm{E}[(x_k - \hat{x}_k)(x_k - \hat{x}_k)^\top]$ |
| $P_k^-$ | **先验**估计误差协方差，$P_k^- = \mathrm{E}[(x_k - \hat{x}_k^-)(x_k - \hat{x}_k^-)^\top]$ |
| $K_k$ | 卡尔曼增益（n×m） |
| $I$ | 单位矩阵（n×n） |

#### 符号命名规则

- **下标 $k$**：第 $k$ 个离散时刻
- **上标 $\;\hat{}\;$（帽子）**：估计值（我们算出来的），不是真实值
  - $x_k$ 无帽子 = 真实状态（上帝视角，算法不可见）
  - $\hat{x}_k$ 有帽子 = 估计状态（算法算出来的）
- **上标 $^-$（减号）**：先验——在看到了 $z_k$ **之前**算出的量
  - $\hat{x}_k^-$ = 纯靠模型猜的状态（还没看传感器）
  - $P_k^-$ = 对那次猜测的不确定程度
- **无上标 $^-$**：后验——在看到了 $z_k$ **之后**修正过的量
  - $\hat{x}_k$ = 看了传感器之后修正过的状态估计
  - $P_k$ = 修正后的不确定程度

#### 方程用真实值，公式用估计值——这俩必须分清

$$
\begin{aligned}
\text{状态方程（描述客观规律）：}\quad & x_k = F x_{k-1} + B u_k + w_k \quad & &\text{← 真实 $x_k$，无帽子} \\[4pt]
\text{观测方程（描述客观规律）：}\quad & z_k = H x_k + v_k \quad & &\text{← 真实 $x_k$，无帽子} \\[4pt]
\text{卡尔曼预测（算法做的）：}\quad & \hat{x}_k^- = F \hat{x}_{k-1} + B u_k \quad & &\text{← 估计 $\hat{x}_k^-$，有帽子} \\[4pt]
\text{卡尔曼更新（算法做的）：}\quad & \hat{x}_k = \hat{x}_k^- + K_k(z_k - H \hat{x}_k^-) \quad & &\text{← 估计 $\hat{x}_k$，有帽子}
\end{aligned}
$$

- $x_k$（无帽子）出现在方程里，描述"世界本来是怎样的"——我们永远不知道它的值
- $\hat{x}_k$（有帽子）出现在算法公式里，描述"我们认为世界大概是怎样的"——这是卡尔曼滤波算出来的
- 观测方程 $z_k = H x_k + v_k$ 不是计算器——你只知道 $z_k$，你不知道 $x_k$，所以我们不能用这个方程反算 $x_k$。卡尔曼滤波的任务就是用 $z_k$ 和 $\hat{x}_k^-$ 一起逼近 $x_k$

### 2.4 H 矩阵详解 — 把状态"翻译"成观测

#### 为什么观测方程里要有 H？

状态方程和观测方程放在一起看，会发现一个不对称的地方：

$$
x_k = F x_{k-1} + B u_k + w_k \qquad \text{状态在"状态空间"（n 维）}
$$
$$
z_k = H x_k + v_k \qquad\qquad\;\; \text{观测在"观测空间"（m 维）}
$$

**状态和观测往往不在同一个空间里**——维度不同、单位不同、物理含义也不同。H 的作用就是在这两个空间之间架一座桥：把高维的状态映射到低维的观测，让"传感器理论上应该读到什么"可以被算出来。

#### 最简单的情况：传感器直接测量某个状态量

当传感器恰好能直接测到状态向量中的某个分量时，H 只有 0 和 1：

```
状态 x = [位置, 速度, 加速度]ᵀ

GPS 只测位置：   H = [1, 0, 0]    "选第一个分量"
轮速计只测速度： H = [0, 1, 0]    "选第二个分量"
IMU 只测加速度： H = [0, 0, 1]    "选第三个分量"
```

这种情况下，H 确实是在"筛选"。但这是特例。

#### H 的核心角色

```
状态空间 (n 维)                    观测空间 (m 维)
"系统真实的样子"                   "传感器眼里看到的样子"

   [x₁]                               
   [x₂]         ──── H ────→         [读数₁]
   [x₃]        "翻译 + 降维"         [读数₂]
   [x₄]                              

n ≥ m（观测维度通常 ≤ 状态维度）
```

| H 的形式 | 含义 | 出现场景 |
|---|---|---|
| 只有 0 和 1 | 传感器直接测到状态分量本身 | GPS 测位置、编码器测速度 |
| 常数（非 0/1） | 传感器测的是状态的线性组合 | 总水量 = 各水位之和 |
| 随时间变化的小数 | 做了线性化 | 雷达测距、超声波测距 |
| 多行多列 | 多种传感器同时融合 | GPS + IMU + 视觉 |

#### H 与残差的关系

公式④中的残差 $z_k - H \hat{x}_k^-$，拆开来看：

```
zₖ：         传感器实际读数       ← "传感器真看到了什么"
H x̂ₖ⁻：      "按我目前的状态估计，传感器理论上应该看到什么"
zₖ - H x̂ₖ⁻： 两者之差             ← "现实"和"预期"之间的 gap
```

**为什么不能直接写 $z_k - \hat{x}_k^-$？** 因为 $z_k$ 是 m 维（比如一个高度读数 2.03），$\hat{x}_k^-$ 是 n 维（比如 [位置, 速度]ᵀ），维度不匹配，物理含义也不一样，根本没法相减。H 把状态估计"投射"到观测空间，让减法两边维度一致、单位一致、物理含义一致。

> **H 不是筛选器，而是翻译器。它回答的问题是："按我们的先验估计 $\hat{x}_k^-$，传感器理论上应该读到什么？"**

### 2.5 P 矩阵详解 — 协方差、椭圆、特征值

#### 2.5.1 从一维方差到多维协方差矩阵

一维估计只有一个方差 $\sigma^2$，描述"估计大概会偏多远"。多维时，仅靠各维度自己的方差不够——**描述不了"x 偏大时 y 是不是也偏大"**。

如果将两个变量的正态分布在几何上表示出来：**形成的区域是一个椭圆**。因为正态分布的性质——变量在均值附近概率最大，离均值越远概率越小。一维形成一条钟形曲线，二维形成一个"小山包"，椭圆就是这个小山包的等高线。

例如状态 $(x, y)$ 的 2×2 协方差矩阵：

$$
P = \begin{bmatrix}
\sigma_x^2 & \sigma_{xy} \\
\sigma_{xy} & \sigma_y^2
\end{bmatrix}
$$

- **对角线**：各维度自己的方差（对每个分量"猜得有多准"的判断）
- **非对角线（协方差）**：两维误差的耦合情况
  - $\sigma_{xy} > 0$ → 正相关，x 偏大 y 也偏大，不确定性椭圆向右上方倾斜
  - $\sigma_{xy} < 0$ → 负相关，x 偏大 y 偏小，椭圆向左下方倾斜
  - $\sigma_{xy} \approx 0$ → 不相关，椭圆摆正

#### 2.5.2 P、Q、R 的区别 — 三个"协方差"各司其职

P、Q、R 都叫"协方差矩阵"，但描述的是不同的东西。用**"来源"和"生命周期"**来区分：

| | P（估计协方差） | Q（过程噪声） | R（观测噪声） |
|---|---|---|---|
| **描述谁？** | 你的**估计**有多不确定 | 你的**模型**有多不准 | 你的**传感器**有多不准 |
| **从哪来？** | 滤波器自己算的（每步变） | 你调的（固定） | 传感器说明书（固定） |
| **谁控制？** | 滤波器（自动） | 你（手动调参） | 传感器厂家 |
| **类比** | "你现在有多迷茫" | "每走一步新产生的迷茫" | "每次睁眼看多清楚" |

**Q 和 R 是"流速"，P 是"水位"：**

```
每步预测：P += Q 的效果    （蒙眼走一步 → 更迷茫）
每次更新：P ×= (1-K) 的效果 （睁眼 → 迷茫消除一部分）

Q 控制水位涨多快，R 控制水位能消到多低
P 是 Q 和 R 拉锯下的动态平衡产物
```

**为什么需要三个？** 因为 Q 和 R 是系统的**固有属性**（不变），P 是滤波器**实时计算**的估计质量（每步变）。没有 Q，P 会一直降到 0，滤波器盲目自信；没有 R，P 会一直涨，滤波器永远不信测量。

#### 2.5.3 P 的特征值与特征向量

$P$ 不是线性变换矩阵，而是描述**不确定性椭圆形状**的矩阵：

| 量 | 几何含义 | 物理含义 |
|---|---|---|
| 特征向量 $\mathbf{v}_1, \mathbf{v}_2$ | 椭圆的长轴方向、短轴方向 | 误差"最不确定的方向"和"最确定的方向" |
| 特征值 $\lambda_1, \lambda_2$ | 沿那个方向的方差大小 | $\lambda$ 越大，那个方向上越不确定 |

**与 3B1B「线性变换」的对应：**

1. 想象一个标准正圆（各方向方差 = 1，无相关性）
2. 存在某变换 $A$，满足 $P = A A^\top$，$A$ 作用在圆上 → 拉成斜椭圆
3. $A$ 的特征向量 = 被拉伸的方向 = $P$ 的特征向量（方向一致）
4. $A$ 的特征值 = 拉伸因子；$P$ 的特征值 = 拉伸因子的平方 = 沿该方向的方差

> $A$ 描述"怎么把圆拉成椭圆"，$P$ 描述"拉完的椭圆长啥样"。两个视角等价。这里用 $A A^\top$ 而非单纯的 $A$，是因为椭圆的长短轴代表的是**方差**（拉伸因子的平方）而非标准差。

### 2.6 矩阵维度速查表 — 所有矩阵的维度从哪来

所有矩阵的维度由两个数字决定：

- **n** = 状态向量的维度（你要估计几个量）
- **m** = 观测向量的维度（传感器测几个量）

#### 维度全量表

| 矩阵 | 维度 | 为什么是这个维度 |
|------|------|-----------------|
| $x_k,\ \hat{x}_k,\ \hat{x}_k^-$ | $n \times 1$ | $n$ 个状态量 |
| $z_k$ | $m \times 1$ | $m$ 个观测量 |
| $F$ | $n \times n$ | 状态到状态，必须方阵 |
| $H$ | $m \times n$ | 把 $n$ 维状态映射到 $m$ 维观测 |
| $P_k,\ P_k^-,\ Q$ | $n \times n$ | 描述 $n$ 个状态的不确定性及两两关联 |
| $R$ | $m \times m$ | 描述 $m$ 个观测量的噪声及两两关联 |
| $K_k$ | $n \times m$ | 把 $m$ 维观测残差分配给 $n$ 个状态修正 |
| $I$ | $n \times n$ | 和 $P$ 维度一致 |

#### 直觉记忆法

```
K 的行 = 你要修正几个状态（n）
K 的列 = 你有几个传感器信号可以用（m）

P 是 n×n：n 个状态，每个都和另外 n-1 个有协方差关系
R 是 m×m：m 个传感器，每个传感器的噪声可能和另一个相关
```

#### 三种常见配置对比

| 场景 | n | m | F | H | P | Q | R | K |
|---|---|---|---|---|---|---|---|---|
| 一维位置跟踪 | 1 | 1 | 1×1 标量 | 1×1 标量 | 1×1 | 1×1 | 1×1 | 1×1 标量 |
| 2D 状态，只测位置 | 2 | 1 | 2×2 | 1×2 行向量 | 2×2 | 2×2 | 1×1 标量 | 2×1 列向量 |
| 2D 状态，测位置+速度 | 2 | 2 | 2×2 | 2×2 | 2×2 | 2×2 | 2×2 | 2×2 矩阵 |
| 4D 状态(x,y,vx,vy)，测 x,y | 4 | 2 | 4×4 | 2×4 | 4×4 | 4×4 | 2×2 | 4×2 矩阵 |

#### K 什么时候是标量/向量/矩阵？

**K 的形状只由 n 和 m 决定：永远是 n×m。**

- n=1, m=1 → K 是标量（一维 KF）
- n=2, m=1 → K 是 2×1 列向量（只测位置，要修正位置+速度两个状态）
- n=2, m=2 → K 是 2×2 矩阵（同时测位置和速度）
- n=4, m=2 → K 是 4×2 矩阵

K 的物理含义：**K[i, j] 表示"第 j 个传感器的偏差，应该分多少去修正第 i 个状态"。** 当 m=1 时 K 只有一列——滤波器自动决定这一个测量偏差如何分配给所有状态。

## 3. 公式推导

因为我本人的数学基础不足，也没有学习过概率论，所以在这一章节只写一些基本的数学推导，请多多海涵。

### 3.1 公式① — 先验状态估计

直接从状态方程来，过程噪声 $w_k$ 的期望为零，所以：

$$
\hat{x}_k^- = F \hat{x}_{k-1} + B u_k
$$

没有任何推导难度——就是把上一时刻的最优估计代入状态方程。

### 3.2 公式② — 先验协方差（$P_k^- = F P_{k-1} F^\top + Q$）

#### 3.2.1 一维直觉

先看标量情况。$F$ 退化为一个数 $f$，$P$ 是方差：

$$
P^- = f^2 \cdot P_{k-1} + Q
$$

**为什么是 $f^2$？**

```
上一时刻：x̂ₖ₋₁，方差 Pₖ₋₁ → 标准差 σ = √Pₖ₋₁
                            → 真实值大概在 x̂ₖ₋₁ ± σ

乘 f 之后：f · x̂ₖ₋₁     → 标准差变成 f·σ = f·√Pₖ₋₁
                         → 方差 = (f·σ)² = f²·Pₖ₋₁
```

方差是平方量，线性因子 $f$ 作用上去，方差乘 $f^2$。写成 $f P f$（对称形式）是为了推广到矩阵做准备。

**为什么 +Q？**

$f^2 P_{k-1}$ 把旧的不确定性传播过来，但模型本身也不是完美的——$w_k \sim \mathcal{N}(0, Q)$ 是这一步的模型误差。两者独立，方差直接相加。

**Q 不会无限叠加吗？**

不会。因为有更新步兜底：

```
预测步：P 涨一点（+Q，"过了一步，更不确定了"）
更新步：P 大幅缩小（看了传感器，"信心恢复了"）

P 在一个范围内震荡，不会无限增长。
```

#### 3.2.2 从一维到多维——一个具体例子

状态 $x = [\text{位置}, \text{速度}]^\top$：

$$
F = \begin{bmatrix} 1 & \Delta t \\ 0 & 1 \end{bmatrix}
\qquad
P_{k-1} = \begin{bmatrix} 4 & 0 \\ 0 & 1 \end{bmatrix}
\qquad
Q = \begin{bmatrix} q_{pos} & 0 \\ 0 & q_{vel} \end{bmatrix}
$$

计算 $F P_{k-1} F^\top$：

$$
\begin{aligned}
F P_{k-1} &= \begin{bmatrix} 1 & \Delta t \\ 0 & 1 \end{bmatrix}
            \begin{bmatrix} 4 & 0 \\ 0 & 1 \end{bmatrix}
            = \begin{bmatrix} 4 & \Delta t \\ 0 & 1 \end{bmatrix} \\[8pt]
(F P_{k-1}) F^\top &= \begin{bmatrix} 4 & \Delta t \\ 0 & 1 \end{bmatrix}
                            \begin{bmatrix} 1 & 0 \\ \Delta t & 1 \end{bmatrix}
                            = \begin{bmatrix} 4 + \Delta t^2 & \Delta t \\ \Delta t & 1 \end{bmatrix}
\end{aligned}
$$

结果解读：

$$
P_k^- = \begin{bmatrix} 4 + \Delta t^2 & \Delta t \\ \Delta t & 1 \end{bmatrix} + Q
$$

| 变化 | 原因 |
|------|------|
| $P_{11}$ 从 4 涨到 $4 + \Delta t^2$ | 新位置 = 旧位置 + 速度×Δt，速度的不确定性混进了位置 |
| $P_{22} = 1$ 不变 | 模型假设速度不变 |
| **$P_{12} = P_{21} = \Delta t$** | 位置和速度现在有了协方差！位置偏大的时候速度也偏大 |

**$F P F^\top$ 做的事：** 不仅传播对角线上的各自方差（$f^2 P$ 的工作），还自动帮你算出了维度之间新产生的耦合（非对角线）。这些耦合仅靠"各自方差"描述不了。

### 3.3 公式④ — 后验状态估计（先讲它，因为③⑤是围绕它推导的）

这是最直观的一个公式——加权平均：

$$
\hat{x}_k = \hat{x}_k^- + K_k (z_k - H \hat{x}_k^-)
$$

| 项 | 含义 |
|---|---|
| $\hat{x}_k^-$ | 先验（模型猜的） |
| $z_k$ | 观测（传感器报的） |
| $z_k - H \hat{x}_k^-$ | **残差（Innovation）**：观测和预测的差距 |
| $K_k$ | **卡尔曼增益**：决定往观测方向修正多少 |

直觉：

- $K \approx 1$ → 后验 ≈ 观测（更信传感器）
- $K \approx 0$ → 后验 ≈ 先验（更信模型）
- $K$ 在 0 和 1 之间 → 两者折中

## 4. 推导逻辑总链

```
我们想得到最优状态估计
       │
       ▼
步骤一：用状态方程做预测
       ① x̂ₖ⁻ = F x̂ₖ₋₁ + B uₖ          （把旧后验代入状态方程）
       ② Pₖ⁻ = F Pₖ₋₁ Fᵀ + Q           （方差经 F 传播，再加模型噪声）
       │
       ▼  得到了先验 (x̂ₖ⁻, Pₖ⁻)
       │
步骤二：用观测来做修正
       设定后验形式：x̂ₖ = x̂ₖ⁻ + K(zₖ - Hx̂ₖ⁻)    ← 加权平均结构
       │
       ▼
步骤三：找最优的 K
       解出 ③ Kₖ = Pₖ⁻ Hᵀ (H Pₖ⁻ Hᵀ + R)⁻¹
       │
       ▼
步骤四：用最优 K 完成更新
       ④ x̂ₖ = x̂ₖ⁻ + Kₖ(zₖ - H x̂ₖ⁻)    （状态修正）
       ⑤ Pₖ = (I - KₖH)Pₖ⁻              （信心恢复）
       │
       ▼  得到了后验 (x̂ₖ, Pₖ)
       │
       └──→ 成为了下一时刻的输入，循环往复
```

## 5. 调参实战

卡尔曼滤波只有两个真正的"旋钮"：Q 和 R。其他矩阵（F、H、P_init）的结构由问题本身决定。

### 5.1 核心原则：Q/R 的比值决定滤波器的"性格"

```
Q/R 大 → 信传感器胜过信模型 → 估计紧跟测量 → 噪声大，但反应快
Q/R 小 → 信模型胜过信传感器 → 估计平滑       → 噪声小，但反应慢
```

### 5.2 第一步：确定 R（查传感器说明书）

R 是传感器的固有属性，不应随便调。如果你有传感器原始数据：

```python
static_readings = np.array([...])  # 传感器静置时的 100 个读数
R = np.var(static_readings)        # 方差 = R
```

如果传感器说明书写"精度 ±3m（1σ）"，那么 $R = 3^2 = 9$。

### 5.3 第二步：调 q（Q 矩阵中唯一的自由参数）

对于匀速模型 $F = \begin{bmatrix} 1 & \Delta t \\ 0 & 1 \end{bmatrix}$，Q 的结构由离散白噪声加速模型推导：

$$
Q = q \cdot \begin{bmatrix} \frac{\Delta t^4}{4} & \frac{\Delta t^3}{2} \\ \frac{\Delta t^3}{2} & \Delta t^2 \end{bmatrix}
$$

只需要调**一个标量 q**——加速度噪声强度。


### 5.4 第三步：看图诊断

跑完仿真，看 KF 估计曲线判断参数好坏：

| 看到的 | 诊断 | 调法 |
|----------|------|------|
| KF 曲线紧跟测量值，抖动大 | Q 太大（信模型不够）| q ↓ |
| KF 曲线平滑但总慢半拍 | Q 太小（信模型太多）| q ↑ |
| 前几步震荡很大才收敛 | P_init 太小（初始太自信）| P_init × 10 |
| 前几步收敛太慢 | P_init 太大 | P_init ÷ 10 |

**P_init 只看前几步，Q/R 看稳态。两者独立。**

### 5.5 2D 的特殊注意：Q[0,0] 天然很小

由于 $Q_{11} = q \cdot \Delta t^4 / 4$，当 $\Delta t = 0.1$ 时 $\Delta t^4 = 0.0001$，Q[0,0] 比 Q[1,1] 小两个数量级。这意味着**位置的过程噪声天然远小于速度的过程噪声**——这是物理上正确的（短时间内随机加速度对速度的影响比对位置的累积影响更直接），但在调参时要注意：想让 K_pos 不收敛到 0，q 需要足够大。

## 6. NumPy 工程踩坑指南

### 6.1 1D 数组 vs 2D 数组

```python
a = np.array([1, 2, 3])          # shape (3,)  — 1D，不是行也不是列
b = np.array([[1, 2, 3]])        # shape (1,3) — 2D 行向量
c = np.array([[1], [2], [3]])    # shape (3,1) — 2D 列向量

a.T      # → 还是 (3,)！.T 对 1D 数组无效！
b.T      # → shape (3,1)，正常转置
```

**卡尔曼滤波里所有矩阵运算都必须用 2D 数组。** H 写 `np.array([[1, 0]])` 而不是 `np.array([1, 0])`。

### 6.2 `*` vs `@`

```python
A * B    # 逐元素乘（Hadamard 积），不是矩阵乘法
A @ B    # 矩阵乘法（Python 3.5+）
```

### 6.3 `K @ H` 可能是点积也可能是外积

当 `H = np.array([[1, 0]])`（shape `(1,2)`），K 自然成为 `(2,1)` 列向量：

```python
K @ H    # (2,1) @ (1,2) = (2,2) 矩阵 ✓  — 外积，公式⑤需要这个
H @ K    # (1,2) @ (2,1) = (1,1) 标量   — 内积（公式③的分母里用到）
```

**但如果 H 是 1D 数组 `(2,)`，K 也会是 `(2,)`，此时 `K @ H` 退化成了点积（标量）。** 解决办法就是把所有参与矩阵运算的数组都写成 2D。

### 6.4 `np.outer()` 作为保险

```python
np.outer(K, H)    # 保证返回外积矩阵，不管 K 和 H 是几维
```

当你不确定 K 和 H 的形状时，`np.outer(K.flatten(), H.flatten())` 永远安全。

### 6.5 随机种子

```python
np.random.seed(42)    # 固定种子 → 每次运行同样的"随机"数 → 方便调试
```

跑对比实验（比如不同 q 值）时务必固定种子，否则噪声不同导致无法公平比较。
